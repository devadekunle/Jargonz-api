# Jargonz — Backend Requirements Specification

## Overview

Jargonz is a vocabulary-building app that helps users capture unfamiliar words from their reading, look up definitions (via LLM), and review them using a spaced repetition system (SRS). The backend provides a RESTful JSON API that powers the frontend React application.

---

## 1. Data Models

### 1.1 Book

Represents a book the user is reading.

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Primary key |
| `title` | string | Book title |
| `author` | string | Author name |
| `coverColor` | string | Hex colour for spine rendering (assigned randomly on creation) |
| `wordCount` | integer | Number of words saved from this book (denormalised, updated via trigger or counter) |
| `createdAt` | ISO 8601 datetime | When the book was added |

### 1.2 WordEntry

A word the user has saved, with SRS metadata.

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Primary key |
| `word` | string | The word itself |
| `bookId` | UUID (FK → Book) | Which book this word came from |
| `bookTitle` | string | Denormalised book title for display convenience |
| `definition` | string | Definition (populated by LLM) |
| `phonetic` | string | Phonetic transcription (populated by LLM) |
| `partOfSpeech` | string | e.g. noun, verb, adjective (populated by LLM) |
| `etymology` | string | Word origin (populated by LLM) |
| `contextSentence` | string | The sentence where the user found the word |
| `exampleSentence` | string | Example usage (generated by LLM) |
| `userNotes` | string | User's personal notes |
| `pageNumber` | integer (nullable) | Page number in the source book |
| `easeFactor` | float | SM-2 ease factor (default 2.5) |
| `interval` | integer | SM-2 interval in days |
| `repetitions` | integer | SM-2 consecutive correct repetitions |
| `nextReviewDate` | date (ISO 8601) | Next scheduled review date |
| `timesReviewed` | integer | Total number of reviews |
| `timesCorrect` | integer | Number of times graded ≥ 3 |
| `masteryLevel` | enum: `learning` / `familiar` / `mastered` | Derived from SRS state |
| `createdAt` | ISO 8601 datetime | When the word was added |

### 1.3 UserProgress

Persistent user level and XP data.

| Field | Type | Description |
|-------|------|-------------|
| `userId` | UUID | Primary key (maps to auth user) |
| `totalXP` | integer | Accumulated experience points |
| `level` | integer | Current level (1–8) |
| `title` | string | Level title (e.g. "Novice", "Scholar") |
| `nextLevelXP` | integer | XP threshold for next level |
| `icon` | string | Emoji icon for current level |
| `updatedAt` | ISO 8601 datetime | Last update timestamp |

**Level thresholds** (defined server-side):

| Level | Title | XP Required |
|-------|-------|-------------|
| 1 | Novice 🌱 | 0 |
| 2 | Apprentice 📜 | 100 |
| 3 | Scholar 📖 | 350 |
| 4 | Bibliophile 📚 | 800 |
| 5 | Lexicographer ✍️ | 1500 |
| 6 | Wordsmith 🔮 | 2800 |
| 7 | Sage 🦉 | 5000 |
| 8 | Luminary 🌟 | 10000 |

### 1.4 ReviewSession (optional, for analytics)

Tracks a completed review session.

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Primary key |
| `userId` | UUID (FK) | |
| `wordsReviewed` | integer | Number of words reviewed |
| `xpEarned` | integer | XP earned in this session |
| `comboPeak` | integer | Highest combo streak |
| `ratings` | JSON array | Array of `{ wordId, grade }` objects |
| `completedAt` | ISO 8601 datetime | |

## 2. API Endpoints

Base URL: `/api/v1`

### 2.1 Books

#### `GET /books`

List all books.

**Response `200`**
```json
[
  {
    "id": "uuid",
    "title": "The Great Gatsby",
    "author": "F. Scott Fitzgerald",
    "coverColor": "#5B4A3A",
    "wordCount": 12,
    "createdAt": "2026-01-15T10:00:00Z"
  }
]
```

#### `GET /books/:id`

Get a single book by ID.

**Response `200`** — single Book object.  
**Response `404`** — `{ "error": "Book not found" }`

#### `POST /books`

Create a new book.

**Request body**
```json
{
  "title": "string (required)",
  "author": "string (required)"
}
```

**Response `201`** — the created Book object (with a randomly assigned `coverColor`).

#### `DELETE /books/:id`

Delete a book and all its associated words.

**Response `204`** — no content.  
**Response `404`** — `{ "error": "Book not found" }`

### 2.2 Words

#### `GET /words`

List all words. Supports optional query parameters.

| Param | Type | Description |
|-------|------|-------------|
| `bookId` | UUID | Filter by book |
| `mastery` | `learning` / `familiar` / `mastered` | Filter by mastery level |
| `search` | string | Full-text search on `word` and `definition` |
| `due` | boolean | If `true`, return only words where `nextReviewDate <= today` |
| `limit` | integer | Pagination limit (default 50) |
| `offset` | integer | Pagination offset (default 0) |

**Response `200`**
```json
{
  "words": [ /* WordEntry[] */ ],
  "total": 42
}
```

#### `GET /words/:id`

Get a single word by ID.

**Response `200`** — single WordEntry object.  
**Response `404`** — `{ "error": "Word not found" }`

#### `POST /words`

Add a new word. The backend should call an LLM to populate `definition`, `phonetic`, `partOfSpeech`, `etymology`, and `exampleSentence`.

**Request body**
```json
{
  "word": "string (required)",
  "bookId": "UUID (required)",
  "contextSentence": "string (optional)",
  "pageNumber": "integer (optional)"
}
```

**Response `201`** — the fully populated WordEntry object (after LLM enrichment).

> **LLM Integration**: The backend should call an LLM (e.g. OpenAI, Claude) with a prompt like:  
> *"Provide the definition, phonetic transcription, part of speech, etymology, and an example sentence for the word '{word}'. Context: '{contextSentence}'."*  
> Parse the response and populate the corresponding fields. If the LLM call fails, return the word with placeholder text and a `"llmStatus": "pending"` flag; a background job can retry.

#### `PUT /words/:id`

Update a word's editable fields.

**Request body** (all fields optional)
```json
{
  "definition": "string",
  "phonetic": "string",
  "partOfSpeech": "string",
  "etymology": "string",
  "contextSentence": "string",
  "exampleSentence": "string",
  "userNotes": "string",
  "pageNumber": "integer"
}
```

**Response `200`** — updated WordEntry object.

#### `DELETE /words/:id`

Delete a word.

**Response `204`** — no content.

#### `GET /words/wotd`

Get the "Word of the Day" — a word due for review (or a random word if none are due).

**Response `200`** — single WordEntry object.

### 2.3 Reviews

#### `POST /reviews`

Submit a review grade for a word. The backend applies the SM-2 algorithm and returns the updated word.

**Request body**
```json
{
  "wordId": "UUID (required)",
  "grade": "integer (required, 0–5)"
}
```

**Grade semantics:**
| Grade | Meaning |
|-------|---------|
| 0 | Complete blackout |
| 1 | Wrong but familiar |
| 2 | Wrong but easy |
| 3 | Difficult but correct |
| 4 | Hesitant correct |
| 5 | Perfect recall |

**SM-2 Algorithm (server-side):**
- If grade < 3: reset `repetitions` to 0, set `interval` to 1, decrease `easeFactor` by 0.2 (min 1.3).
- If grade ≥ 3:
  - 1st correct: interval = 1
  - 2nd correct: interval = 6
  - Subsequent: interval = round(interval × easeFactor)
  - Update easeFactor: `EF' = EF + (0.1 - (5 - grade) × (0.08 + (5 - grade) × 0.02))`, min 1.3
- Set `nextReviewDate` to today + interval days.
- Increment `timesReviewed`; increment `timesCorrect` if grade ≥ 3.
- Derive `masteryLevel`:
  - `repetitions >= 5 && easeFactor >= 2.3` → `mastered`
  - `repetitions >= 2` → `familiar`
  - otherwise → `learning`

**Response `200`**
```json
{
  "wordId": "uuid",
  "grade": 4,
  "updatedEntry": { /* full WordEntry with new SRS state */ }
}
```

#### `POST /reviews/session`

Submit a batch of reviews (entire session at once). Useful for offline resilience.

**Request body**
```json
{
  "ratings": [
    { "wordId": "uuid", "grade": 4 },
    { "wordId": "uuid", "grade": 5 }
  ]
}
```

**Response `200`**
```json
{
  "results": [ /* ReviewResult[] */ ],
  "xpEarned": 12,
  "comboPeak": 3
}
```

  "title": "Scholar",

Add XP to the user's account. Called when a review session completes.

**Request body**
```json
{
  "amount": 12
}
```

**Response `200`**
```json
{
  "progress": { /* UserProgress object */ },
  "didLevelUp": false,
  "previousLevel": 3
}
```

**XP earning rules** (enforced server-side):

| Grade | Base XP | Notes |
|-------|---------|-------|
| 1 (Again) | 0 | No XP for forgotten words |
| 3 (Hard) | 2 | |
| 4 (Good) | 4 | |
| 5 (Easy) | 6 | |

**Combo bonus**: For every 5 consecutive correct answers (grade ≥ 4), award +1 bonus XP. The combo counter resets on any grade < 4.

## 3. Authentication

- Use **JWT-based authentication**.
- `POST /auth/register` — create account (email + password).
- `POST /auth/login` — returns JWT access token + refresh token.
- All endpoints except `/health`, `/auth/register`, and `/auth/login` require `Authorization: Bearer <token>` header.
- Token expiry: 24 hours for access token, 30 days for refresh token.
- `POST /auth/refresh` — exchange refresh token for a new access token.

## 4. LLM Integration

When a word is added via `POST /words`, the backend should:

1. Immediately return a `202 Accepted` with the word in a pending state, OR
2. Block and call the LLM synchronously, returning the enriched word on `201`.

**Recommended approach**: Synchronous with a timeout fallback.

**LLM prompt template**:
```
You are a lexicographer. For the word "{word}"{context_clause}, provide:
1. Definition (concise, 1-2 sentences)
2. Phonetic transcription (IPA)
3. Part of speech
4. Etymology (brief)
5. An example sentence showing proper usage

Respond in JSON format with keys: definition, phonetic, partOfSpeech, etymology, exampleSentence
```

Where `{context_clause}` is ` found in the context: "{contextSentence}"` if a context sentence was provided.

## 5. Error Handling

All errors should return a consistent JSON structure:

```json
{
  "error": "Human-readable error message",
  "code": "ERROR_CODE_STRING",
  "details": {} // optional, additional context
}
```

**Standard HTTP status codes:**
| Code | When |
|------|------|
| 200 | Success |
| 201 | Created |
| 204 | No content (deletion) |
| 400 | Bad request (validation error) |
| 401 | Unauthorized (missing/invalid token) |
| 404 | Resource not found |
| 409 | Conflict (e.g. duplicate word) |
| 429 | Rate limited |
| 500 | Internal server error |

---

## 6. Rate Limiting

- 100 requests per minute per user for general endpoints.
- 10 requests per minute for `POST /words` (to prevent LLM abuse).
- Return `429 Too Many Requests` with `Retry-After` header.

## 8. Database Schema (PostgreSQL)

```sql
CREATE TABLE books (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL,
  author TEXT NOT NULL,
  cover_color TEXT NOT NULL DEFAULT '#5B4A3A',
  word_count INTEGER NOT NULL DEFAULT 0,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE words (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  word TEXT NOT NULL,
  book_id UUID NOT NULL REFERENCES books(id) ON DELETE CASCADE,
  book_title TEXT NOT NULL,
  definition TEXT NOT NULL DEFAULT '',
  phonetic TEXT NOT NULL DEFAULT '',
  part_of_speech TEXT NOT NULL DEFAULT '',
  etymology TEXT NOT NULL DEFAULT '',
  context_sentence TEXT NOT NULL DEFAULT '',
  example_sentence TEXT NOT NULL DEFAULT '',
  user_notes TEXT NOT NULL DEFAULT '',
  page_number INTEGER,
  ease_factor REAL NOT NULL DEFAULT 2.5,
  interval INTEGER NOT NULL DEFAULT 0,
  repetitions INTEGER NOT NULL DEFAULT 0,
  next_review_date DATE NOT NULL DEFAULT CURRENT_DATE,
  times_reviewed INTEGER NOT NULL DEFAULT 0,
  times_correct INTEGER NOT NULL DEFAULT 0,
  mastery_level TEXT NOT NULL DEFAULT 'learning'
    CHECK (mastery_level IN ('learning', 'familiar', 'mastered')),
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_words_book_id ON words(book_id);
CREATE INDEX idx_words_next_review ON words(next_review_date);
CREATE INDEX idx_words_mastery ON words(mastery_level);
CREATE INDEX idx_words_search ON words USING gin(to_tsvector('english', word || ' ' || definition));

CREATE TABLE user_progress (
  user_id UUID PRIMARY KEY,
  total_xp INTEGER NOT NULL DEFAULT 0,
  level INTEGER NOT NULL DEFAULT 1,
  title TEXT NOT NULL DEFAULT 'Novice',
  next_level_xp INTEGER NOT NULL DEFAULT 100,
  icon TEXT NOT NULL DEFAULT '🌱',
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE review_sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL,
  words_reviewed INTEGER NOT NULL DEFAULT 0,
  xp_earned INTEGER NOT NULL DEFAULT 0,
  combo_peak INTEGER NOT NULL DEFAULT 0,
  ratings JSONB NOT NULL DEFAULT '[]',
  completed_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_review_sessions_user ON review_sessions(user_id);
CREATE INDEX idx_review_sessions_date ON review_sessions(completed_at);
```

## 9. Future Considerations (out of scope for MVP)

- **Offline support**: Sync queue for reviews created offline.
- **Social features**: Share words, follow users, leaderboards.
- **Import/Export**: Export word list as CSV/Anki deck.
- **Multiple languages**: Support for non-English books and words.
- **Audio pronunciations**: TTS integration for phonetic playback.
- **Book metadata**: Auto-fill author/cover from ISBN lookup.
